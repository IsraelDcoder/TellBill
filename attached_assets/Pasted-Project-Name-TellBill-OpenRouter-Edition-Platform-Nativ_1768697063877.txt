Project Name: TellBill (OpenRouter Edition)
Platform: Native Mobile (iOS & Android)
Tech Stack: Flutter (Frontend), Supabase (Backend/Auth/DB/Storage), Node.js (Server), OpenRouter (AI), Stripe (Payments)

Goal:
Build a fully functional enterprise-grade contractor voice-first invoicing app that is monetizable from day one. No mock data. Everything must work, including AI, PDFs, payments, and subscription logic.

2. AUTHENTICATION & ORGANIZATION (RBAC)
---------------------------------------------------------------------
- Supabase Auth (email/password)
- Organization model
- Roles:
  - Admin (owner)
  - Worker (contractor)
- Workers only see their own logs/invoices
- Admin sees company-wide analytics and team activity

---------------------------------------------------------------------
3. OFFLINE-FIRST + SECURITY
---------------------------------------------------------------------
- Local storage via SQLite for offline usage
- Sync when connection is stable
- Conflict resolution (last write wins + merge)
- AES-256 encryption for voice recordings before upload
- Audit log: every invoice generation logged with timestamp + GPS

---------------------------------------------------------------------
4. AI PIPELINE (OpenRouter)
---------------------------------------------------------------------
Module A: Voice Recording
- One-tap “Record” button
- Save audio locally (encrypted)
- Upload audio to server for transcription

Module B: Transcription (OpenRouter)
- Use open-source STT model through OpenRouter
- Show live transcription
- Noise suppression enabled

Module C: Reasoning (OpenRouter LLM)
- Send transcription to LLM via OpenRouter
- Extract structured JSON:
  - client_name
  - materials_list (name, qty, unit price)
  - labor_hours
  - job_address (GPS)
  - safety_notes
  - payment_terms
  - tax_rate
- AI confidence score
  - <85% -> highlight fields for manual correction
